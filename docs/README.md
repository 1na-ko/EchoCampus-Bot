## 📝 文档维护规范

### 文档命名规范
- 使用中文命名，清晰表达文档用途
- 过期文档添加 `_旧` 后缀或移至 `archive/` 目录
- 版本化文档使用 `_v1`、`_v2` 等后缀

### 更新时效标记
所有文档应在开头标注：
```markdown
> **最后更新**: YYYY年MM月DD日
```

### 文档分类原则
| 目录 | 用途 | 示例 |
|------|------|------|
| `deployment/` | 部署、运维相关 | 部署指南、环境配置 |
| `development/` | 开发、测试相关 | 测试指南、编码规范 |
| `reference/` | 技术参考资料 | 数据库设计、API规范 |
| `archive/` | 历史归档文档 | 审查报告、迁移指南 |
